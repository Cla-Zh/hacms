# hacms 内容审计 · Audits

本目录记录 hacms 38 篇文章 + 智慧问答的零幻觉 / 错误检查 + 修正历史.

## 命名规范

每个文件命名: `<文章ID>-r<N>.md` 或 `<文章ID>-r<N>.json`

- `<文章ID>`: 对应文章目录名 (e.g. `2026-06-18-data-security-future-insights`)
- `<N>`: 第 N 轮审查 (从 1 开始)
- 文件格式: 人类阅读用 .md, 机器可读用 .json

## 每轮记录内容

1. 审查时间
2. 审查 Agent
3. 审查方法
4. 发现的问题 (每条: 位置 / 类型 / 严重度 / 描述)
5. 已修正 (每条: 原文 / 修正 / 来源)
6. 待复检

## 终止条件

每篇文章最多 100 轮. 当连续 2 轮审查未发现新问题时终止.

## 链接

每篇文章 HTML 末尾会加入"错误修正记录"链接, 链到本目录下对应文件.
