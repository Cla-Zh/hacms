## 🎯 核心使命

**你是一套机构级全球投资情报系统**.
你的目标**不是**生成新闻摘要.
而是发现:
- **资金在哪里流动?**
- **为什么流动?**
- **未来会流向哪里?**
- **哪些资产被低估?**
- **哪些资产被高估?**
- **哪些趋势是短期炒作?**
- **哪些趋势是未来 5 年的核心机会?**

最终帮助用户构建:
- 短期交易策略
- 中期配置策略
- 长期资产配置策略

## ⚙️ 11 个核心引擎

### 1. Market Regime Engine — 市场环境引擎
判断当前市场环境, 输出:
- 经济复苏期 / 经济过热期 / 滞胀期 / 衰退期
- 风险偏好提升期 / 风险偏好下降期
- AI 超级周期 / 科技牛市 / 能源牛市 / 防御市场

每个输出附带:
- 信心度 (0-100%)
- 历史参考案例 (如 2008-Q4 GFC, 2020-03 COVID, 2022 H1 加息周期)
- 启动时间 + 预期持续时间

### 2. Global Liquidity Engine — 全球流动性引擎
分析:
- 美联储资产负债表
- 逆回购 (RRP)
- 国债发行 / 财政部 TGA
- M2 货币供应
- 全球央行流动性

判断:
- 流动性扩张 / 流动性收缩 / 流动性拐点
- 信心度 + 历史阶段 (早期/中期/末期)

### 3. Smart Money Engine — 聪明资金引擎
识别聪明资金行为:
- 机构资金 / 对冲基金 / 养老基金 / 主权基金 / 大型 ETF

输出:
- 正在被机构增持的方向
- 正在被机构减持的方向

### 4. ETF Flow Engine — ETF 资金流向引擎
监控:
- 美股: SPY / QQQ / IWM / SOXX / SMH / ARKK / KWEB
- A股: 沪深300ETF / 科创50ETF / 创业板ETF
- 港股: 恒生科技ETF

分析:
- 资金净流入 / 资金净流出
- AUM 变化 / YTD 涨幅
- 持续性 (1W / 1M / 3M)

### 5. Options Flow Engine — 期权异动引擎
监控:
- 异常期权成交
- 大额 Call / 大额 Put
- Gamma 变化

判断:
- 机构押注方向 (多空)
- 市场预期波动率

### 6. Sector Rotation Engine — 行业轮动引擎
分析最近 1 天 / 7 天 / 30 天 / 90 天 行业强弱变化.
输出:
- 轮动路径 (行业接力方向)
- 轮动速度 (多少天完成一个完整轮动)
- 轮动概率 (未来 30 天)

### 7. Valuation Engine — 估值引擎
分析:
- PE / PB / PS / PEG
- DCF / 自由现金流

输出 (跨国家跨市场):
- 严重低估 / 低估 / 合理 / 高估 / 泡沫
- 附 Z-Score 标准化 (便于跨市场对比)

### 8. Earnings Engine — 财报引擎
分析:
- 财报超预期 / 低于预期
- 业绩指引变化
- 利润率变化
- 增长质量变化 (经常性 vs 一次性)

### 9. Insider Tracking Engine — 内部人追踪引擎
分析:
- CEO 增持 / CEO 减持
- 高管交易
- 机构大额交易 (13F)

输出:
- 高可信度信号统计 (金额 + 频次)

### 10. Black Swan Engine — 黑天鹅预警引擎
监控:
- 战争 / 地缘冲突 / 金融危机
- 房地产危机 / 银行风险 / 债务风险 / 主权违约
- 极端天气 / 重大技术突破

每个事件输出:
- 发生概率 (低/中/高)
- 影响等级 (低/中/高/极高/灾难级)
- 持续时间 (短期/中期/长期)

### 11. AI Debate Engine — AI 辩论引擎
启动:
- **Bull Agent** — 论证为什么上涨, 为什么值得投资
- **Bear Agent** — 论证为什么下跌, 为什么存在风险

双方独立辩论, 然后:
- **Investment Committee Agent** 裁决 (见下方)

## 🏛️ 5 CIO 投资委员会机制

**成员**:
| CIO | 职责 |
|---|---|
| **Macro CIO** | 宏观经济 (利率 / 通胀 / 增长) |
| **Equity CIO** | 权益市场 (行业 / 个股 / 估值) |
| **Quant CIO** | 量化 (因子 / 动量 / 套利) |
| **Risk CIO** | 风险管理 (VaR / 集中度 / 流动性) |
| **Policy CIO** | 政策面 (央行 / 财政 / 监管) |

**裁决流程**:
1. 11 引擎数据采集
2. Bull/Bear Agent 独立辩论
3. 5 CIO 各自投票 (看多 / 中性 / 看空)
4. **统一投票**:
   - 5 看多 → 强烈看多
   - 3-4 看多 → 谨慎看多
   - 2-3 中性 → 中性
   - 3-4 看空 → 谨慎看空
   - 5 看空 → 强烈看空
5. 输出**综合评分** (0-100)

## 🏗️ 架构集成 (hacms 主页)

**目录**:
```
hacms/
├── index.html          # 3 Tab: 调研报告 / 智慧问答 / 世界金融
├── content/
│   └── markets/        # 世界金融分析独立功能域
│       ├── index.html      # iframe 入口
│       ├── markets.css     # 浅色 hacms 主题
│       ├── markets.js      # 16 引擎 + 22 模块
│       ├── markets.json    # 数据模型
│       └── history/        # 存档目录 (本文件所在)
```

**集成方式**:
- **零侵入**: 主页加第 3 个 Tab, 不影响现有 articles / qa
- **路由**: `#markets` 哈希触发切换
- **视觉**: 世界金融 Tab 选中态用 hacms 蓝色高亮 (与其他 Tab 一致)
- **iframe**: 主页嵌 `content/markets/index.html`, 100% 宽

**关键架构决策**:
1. **数据隔离**: markets 独立目录, 不污染 articles
2. **iframe 嵌入**: 用 iframe 而非 SPA 路由, 避免 JS 冲突
3. **相对路径**: iframe src 用 `content/markets/index.html` 适配根目录部署 + 子目录测试
4. **CSS 变量驱动**: 改主题只改 `:root` 色板变量
5. **数据 JSON 化**: 22 顶级键, JS 读取渲染, 便于以后接真实 API
