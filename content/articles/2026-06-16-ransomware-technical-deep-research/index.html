4. 加密模式详解（全文件 / 部分 / 间歇）

4.1 三种加密模式定义

📌 Full File Encryption（全文件加密）

加密文件的全部内容。耗时长、破坏彻底，是早期勒索病毒的主流模式。LockBit 2.0/3.0、Conti、REvil、DarkSide、RansomEXX、Medusa 默认采用此模式。

📌 Partial File Encryption（部分文件加密）

仅加密文件的特定偏移或固定比例。典型如 BianLian——Avast 静态分析明确指出"不从文件开头/结尾加密"，存在固定文件偏移（offset），从该偏移处开始加密（Avast Decryptor）。

📌 Intermittent Encryption（间歇加密）

每隔 N 字节加密一段，跳过中间字节。SentinelLabs 2022-09 报告称"勒索开发者转向间歇加密以逃避检测"。典型代表：

  • Babuk：公开源码支持间歇加密（BleepingComputer）
  • BlackCat：通过配置参数决定每 N 字节加密一次（Sentinel LABS）
  • Play：典型实现，间隔大小因 build 而异
  • LockFile：每 16 字节加密、跳过 16 字节（Sophos 描述）

4.3 为什么勒索病毒越来越多使用间歇加密？

SentinelLabs 2022-09 报告核心论点（原文链接）：

  1. 逃避熵检测：传统 EDR 通过文件熵值突变判断加密行为；间歇加密的"加密 + 跳过"模式让文件整体熵值仍接近原始文件
  2. 加速加密：跳过 N 字节可节省 50% 以上的 I/O 时间（视具体比例）
  3. 破坏仍然有效：即便只加密文件 50% 字节，文件也已损坏无法正常使用（多数格式需要完整字节流）
  4. 绕过卡方检验：Sophos 对 LockFile 做过分析，其间歇加密的输出通过卡方检验，统计上无法被识别为"加密"

5. 数据窃取与外泄机制

5.1 双重勒索（Double Extortion）模式

双重勒索的最早大规模实现可追溯到 2019 年末 Maze 勒索团伙，被公认为"第一个公开在专设网站上泄露未付款受害者数据的团伙"。

双重勒索的执行机制可分解为以下环节（基于 Sekoia TDR 2024-11-27）：

  1. Collection（数据收集）：通过 LOLBin、AdFind、PowerShell 等在受害网络中识别高价值文件
  2. Staging（数据暂存）：将文件集中到本地临时目录（如 C:\Users\Public、PerfLogs）
  3. Compression & Encryption（压缩加密）：使用 7-Zip、WinRAR 等打包；有些团伙对档案加密
  4. Exfiltration（外泄）：通过合法或自定义工具上传
  5. Extortion（勒索）：在 Tor 泄露站点公开施压

7. 加密速度优化技术

7.1 五大核心技术

① I/O Completion Ports (IOCP) 异步 I/O

LockBit 2.0 的核心加速技术（ThreatMon 逆向）：

  1. CreateFile() 时使用 FILE_FLAG_NO_BUFFERING 标志，绕过 Windows 文件缓存
  2. 文件 I/O 全部转为 Native API（NtCreateFile、NtWriteFile）
  3. 异步文件 I/O（ReadFileEx / WriteFileEx）
  4. I/O Completion Ports (IOCP)——Windows 内核级异步 I/O 模型
  5. 直接调用 KiFastSystemCall 进入内核态

② 多线程加密

Conti 实现（详细逆向）：

  • 启动时调用 GetNativeSystemInfo 获取 CPU 核数
  • 默认线程数 = CPU 核数
  • 命令行参数 -m all 时 线程数 = 2 × 核数
  • 每文件独立 256-bit ChaCha8 密钥 + 64-bit nonce

AvosLocker 实现（Zscaler ThreatLabz）：

  • 控制台参数 concurrent_threads_num_max: 200（最高 200 并发）
  • 加密 D:/A: 等小卷 < 1 秒，C: 盘 19.6 秒

③ Windows Restart Manager API

CrowdStrike 2023 年技术博客详细分析：

正常勒索病毒遇到被锁文件会失败。Conti 泄露源码揭示其使用 Rstrtmgr.dll 的 KillFileOwner() 函数：

  1. 动态加载 Rstrtmgr.dll
  2. RmStartSession() 创建会话
  3. 遍历系统所有文件
  4. RmRegisterResources() 注册文件
  5. RmGetList() 两次（第一次取计数，第二次取进程信息）
  6. 与进程白名单比对
  7. RmShutdown() 终止非白名单进程、释放文件锁

Conti 进程白名单（绝对不杀）：

spoolsv.exe, explorer.exe, sihost.exe, fontdrvhost.exe, cmd.exe,
dwm.exe, LogonUI.exe, SearchUI.exe, lsass.exe, csrss.exe,
smss.exe, winlogon.exe, services.exe, conhost.exe

关键优势：RmGetList() 返回的是用户友好名（Description 字段），不依赖可执行文件名，因此能杀掉被改名的安全工具。

④ NTFS Alternate Data Streams (ADS)

Talos 报告：BlackCat 利用 ADS 隐藏持久化：

# BlackCat 把 reverse-ssh 写入 C:\ 的 NTFS ADS 中
get-content c:\system -raw | set-content c:\ -stream 'cachetask'

# 通过任务计划程序以 ADS 中的载荷触发
schtasks /create /ru system /sc minute /tn microsoft\windows\wininet\cachetask /tr c:\:cachetask -b <bind port> /f

文件名格式 c:\:cachetask 表示访问 c:\ 的 cachetask 流——绕开普通 dir 列举。

⑤ 内存映射 I/O（Memory-Mapped I/O）

LockFile / WastedLocker / Maze 使用（Sophos）：

  • CreateFileMapping() 把文件映射到内存
  • 在内存中加密文件内容
  • OS 写回磁盘时无明显磁盘 I/O 模式——绕过基于 I/O 行为的检测

7.2 文件排除白名单（避免 BSOD）

勒索病毒通常排除系统关键文件以避免 BSOD——攻击者需要系统保持运行来展示赎金弹窗、留出加密时间窗：

AvosLocker 排除清单（Zscaler ThreatLabz 详细逆向）

排除目录：

All Users, AppData, boot, bootmgr, Games, Intel, Microsoft.,
Program Files, ProgramData, Public, Sophos, System Volume Information,
Windows, Windows.old, WinNT, "MicrosoftSQL Server"

排除文件：

Thumbs.db, ntuser.ini, ntuser.dat.log, ntuser.dat, ntldr, iconcache.db,
desktop.ini, config.msi, bootsect.bak, bootfont.bin, boot.ini, autorun.inf

为什么排除？

  • 加密 pagefile.sys、hiberfil.sys → 系统无法分页/休眠
  • 加密 C:\Windows\ntoskrnl.exe → 立即 BSOD (CRITICAL_PROCESS_DIED)
  • 加密 %APPDATA% 中运行中的文件 → 应用程序崩溃引发连锁 BSOD
