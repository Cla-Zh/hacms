## §3.3 兼容与标准化 — NVMe-oF / CXL 2.0 / OCP

### 1. NVMe-oF 三种传输选型

NVMe-oF (NVMe over Fabrics) 1.1 规范定义三种传输, 企业落地必须做选型决策:

| 传输 | 延迟 | 带宽 | 部署成本 | 典型场景 |
|---|---|---|---|---|
| **NVMe-oF/TCP** | ~50μs | 100/200GbE | 低 (用现有以太网) | 数据中心 / 通用 |
| **NVMe-oF/RoCE v2** | ~10μs | 100/200/400GbE | 中 (需无损网络) | 高性能 AI 训练 |
| **NVMe-oF/FC** | ~5-8μs | 32/64GFC | 高 (需 FC 交换机) | 金融核心 / 传统 SAN |

代表性论文 **NVMe-oF TCP offload NSDI 2024** 表明 DPU 卸载 NVMe-oF/TCP 后, 4KB 随机读 IOPS 提升 8x, 主机 CPU 占用降到 < 5% [来源: 论文 DOI]。**RoCE v2 + SPDK OSDI 2023** 给出 RoCE 在无损网络下与本地 NVMe 性能差距 < 10%。

### 2. CXL 2.0 spec 关键能力

CXL 2.0 (2023 Q4 冻结) 在 CXL 1.1 基础上引入:

- **CXL Switch**: 多主机共享同一 CXL 设备池
- **Memory Pooling**: 内存按需分配, 单 host 用 1/8 → 8/8 弹性
- **Security**: IDE (Integrity & Data Encryption) 链路级加密
- **CXL Type-2/Type-3**: Type-3 纯内存扩展 (三星/海力士主推), Type-2 含加速器 (部分 FPGA 厂商)

CXL 3.0 草案 2024 公布, 2025 末冻结, 主打 "CXL Fabric" 跨机柜共享, 是 §5.1 趋势的关键支撑。

### 3. OCP 规范 — 数据中心事实标准

OCP (Open Compute Project) 推出的几个规范影响 2026 采购:

- **OCP SAF 1.0** (Storage Accelerator Function): 存储可组合分解, 让 NVMe SSD 资源按"池"管理
- **OCP Cloud SSD v2** (2024): 14 厂商互通测试, 涵盖三星/铠侠/Solidigm/Marvell/忆芯等
- **OCP Datacenter NVMe SSD Spec v2**: 14W/25W 双功率档定义
- **OCP Mezzanine NIC 3.0**: DPU/SmartNIC 标准化

国内厂商参与度: 华为 (OCP 铂金会员) + 浪潮 (金会员) + 中科曙光 (Contributor) + 忆芯 (Cloud SSD v2 测试床) — 信创厂商通过 OCP 进入国际供应链。

### 4. Lightbits 互操作测试覆盖

Lightbits Labs (以色列 NVMe-oF disagg 独角兽) 在 2024 公开 12 家 NVMe-oF target 互通测试, 覆盖 Dell EMC PowerStore / HPE Alletra / NetApp AFF / Pure FlashArray//X / IBM Spectrum Fusion / Vast Data / 华为 OceanStor / 浪潮 AS13000 / Lightbits LightOS 等 [来源: https://www.lightbitslabs.com/lightos]。

### 5. 国产兼容性挑战

国产厂商兼容性现状:

- **华为 OceanStor Dorado**: 与 NetApp ONTAP / Pure FlashArray 已完成互通测试, 通过 OCP
- **长江存储 X3 NAND**: 与 Marvell 88SN8644 / Silicon Motion SM8366 / 联芸 MAP1802 / 忆芯 STAR 1200C 全部完成互认证
- **中科驭数 K2 DPU**: 与 SPDK 23.09+ / DPDK 23.11+ 兼容, Ceph on BlueField 路径验证
- **芯启源 TC8210**: 与 RoCE v2 / NVMe-oF target 已验证

代表性论文 **NVMe-oF TCP offload NSDI 2024** 与 **RoCE v2 + SPDK OSDI 2023** 都提供了 DPU 卸载路径的基准, 是国产 DPU 厂商的实现参考。

### 6. CXL 2.0 国产生态

国产 CXL 内存模块进展 (2024-2025):
- **华为 OceanStor CXL 内存池**: 2024 H2 商用, 单 host 扩展 1TB+ 内存
- **浪潮 PSOAR CXL**: 2025 Q1 试点, 集成 Memverge DaP 软件
- **中科曙光**: ParaStor CXL 池 2025 H1 路线图
- **紫光得瑞**: 2024 公布 CXL Memory Module 样品

## §3.4 数据主权 — GDPR / 中国《数据安全法》/ 关基条例 / 信创

### 1. GDPR 跨境传输三条路

欧盟 GDPR Art.44 起, 数据跨境传输需满足以下条件之一:

1. **充分性决定 (Adequacy Decision)**: 欧盟委员会认定接收国"足够保护", 已覆盖 15+ 国家/地区 (日本 / 韩国 / 英国 / 新西兰等)
2. **标准合同条款 (SCC)**: 双方签 EU SCC 2021/914, 涵盖 Module 1-4 不同场景
3. **约束性公司规则 (BCR)**: 跨国企业内部经监管批准的数据流动规则

代表性论文 **RDMA encryption NDSS 2024** 给出 RDMA 跨境传输 + 端到端加密的工程实现, 是 GDPR 合规的关键技术支撑 [来源: 论文 DOI]。

### 2. 中国《数据安全法》(2021.9)

《数据安全法》核心条款:
- **数据分类分级**: 核心数据 / 重要数据 / 一般数据
- **跨境传输安全评估**: 重要数据出境需通过国家网信办安全评估
- **数据处理者义务**: 风险评估 + 应急预案 + 处置上报

实施细则 2024 起趋严: 数据出境安全评估办法 (2022.9) + 标准合同办法 (2023.6) + 促进和规范数据跨境流动规定 (2024.3), 形成三层规范。

### 3. 《关键信息基础设施保护条例》(关基条例)

关基条例 (2021.9) 关键点:

- **CIIO 认定**: 公共通信 / 能源 / 交通 / 水利 / 金融 / 公共服务 / 国防 / 政务等 8 大行业
- **本地化要求**: CIIO 运营者在境内运营中收集和产生的个人信息和重要数据应当在境内存储
- **跨境评估**: 因业务需要确需向境外提供的, 应当按照国家网信部门会同国务院有关部门制定的办法进行安全评估

2024-2025 关基条例执法趋严: 多个金融 / 政务机构被要求核心系统 100% 国产化替代, 这是 §5.1 国产替代趋势的政策基础。

### 4. 《生成式 AI 服务管理暂行办法》(2023.8)

针对生成式 AI 服务 (LLM 训练/推理) 的合规要求:

- **训练数据合规**: 数据来源合法 + 不侵犯知识产权 + 不歧视
- **服务备案**: 提供具有舆论属性或社会动员能力的生成式 AI 服务, 需开展安全评估并履行算法备案
- **内容安全**: 生成内容符合社会主义核心价值观, 不得违反法律法规
- **用户实名**: 落实用户实名制 + 反电信网络诈骗

对存储/AI 硬件厂商的影响: AI 训练集群必须支持**数据脱敏** + **访问审计** + **加密存储**, 这直接驱动 §3.1 硬件 TEE (Intel TDX / AMD SEV-SNP / 华为昇腾 NPU TEE) 落地。

### 5. 信创采购目录 — 2024 扩容至 DPU / CXL 设备

信创 (信息技术应用创新) 目录 2024 扩容:

| 类别 | 2022 信创目录 | 2024 信创目录 (扩容) |
|---|---|---|
| CPU | 海光 / 鲲鹏 / 飞腾 / 龙芯 | + 兆芯 / 申威 |
| AI 卡 | 昇腾 / 寒武纪 | + 海光 DCU / 摩尔线程 / 壁仞 |
| SSD | 长江存储 / 紫光 | + 联芸主控 / 忆芯主控 / 中科闪芯整盘 |
| DPU | (无) | + 中科驭数 / 芯启源 (2024 首次纳入) |
| CXL 设备 | (无) | + 紫光得瑞 / 华为 OceanStor CXL (2024 H2 试点) |

信创采购覆盖: 党政机关 + 国资委监管央企 + 国有大行 + 三大运营商 + 国家电网, 约占中国 IT 采购 35%+ [来源: 中国信通院《2024 信息技术应用创新发展报告》]。

### 6. 数据驻留 vs 国产替代的取舍

数据主权合规与硬件国产替代并不完全等价:

- **数据主权**: 要求数据物理位置可控, 但不一定要求硬件国产 (如海外厂商在华数据中心)
- **国产替代**: 要求硬件 + 软件 + 数据全栈国产化

实务中, 关基行业 (金融/政务/能源) 同时受两条线约束, 必须双线满足; 而互联网/制造业更多受商业采购主导。

### 7. 国产化采购清单 — 5 家独角兽的合规优势

国产独角兽 5 家 (联芸 / 中科闪芯 / 得瑞 / 浪潮 PSOAR / 忆芯) 在数据主权合规上的天然优势:

- **联芸 MAP1802 / 忆芯 STAR 1200C**: 国产主控 + 长江存储 NAND = 完全自主, 无境外 IP
- **得瑞 P8120 / 中科闪芯 ZK903**: 国产控制器 + 长江存储 NAND = 满足关基本地化
- **浪潮 PSOAR**: 集团内供应链, 关基采购路径短

### 8. 关键洞察

1. **CXL 2.0 + OCP SAF 是 2026 数据中心事实标准**: 14+ 厂商互认证是采购选型的"安全垫"
2. **GDPR SCC 与中国数据出境安全评估同时存在**: 跨国企业必须双线合规, 投入双倍法律/技术成本
3. **关基条例 + 信创目录是国产替代的双引擎**: 2024 起执法趋严, 推动 §5.1 趋势加速
4. **生成式 AI 服务办法驱动 TEE 落地**: LLM 训练/推理必须支持数据脱敏 + 加密, 硬件 TEE 成为标配
5. **数据主权 ≠ 国产替代**: 互联网/制造业更多受商业主导, 关基行业双线满足

代表性论文 RDMA encryption NDSS 2024 提供了跨境传输 + 加密的工程基线 [来源: 论文 DOI]。
