一、核心发现摘要

发现一：训练基础设施已进入"AI Factory"时代——软硬协同设计成为新范式

NVIDIA DGX SuperPOD、Google TPU Pod、Meta 35,000+ H100集群、Tesla Dojo代表四种不同的AI Factory路线：通用超级计算（NVIDIA）、自研芯片+框架协同（Google TPU+Pathways+JAX）、开源极限扩展（Meta PyTorch）、领域定制晶圆级封装（Tesla Dojo）。核心趋势：芯片、互联、框架的联合优化（Co-design）比单点突破更能提升训练效率。

NVIDIA DGX SuperPOD扩展至数千GPU，提供"交钥匙"AI Factory方案，支持兆级参数模型训练。nvidia.com/en-us/data-center/dgx-superpod

发现二：经典模型与Transformer长期共存——混合架构是生产现实

Tesla Autopilot从HydraNet（CNN）演进到BEV+Transformer，但量产车型CNN backbone至今仍在使用；Google Translate从LSTM迁移到Transformer经历了三年渐进式混合架构；字节推荐系统Wide&Deep+实时更新仍在支撑日均500亿次推荐请求。洞察：不是"谁替代谁"，而是各取所长——CNN的推理效率、Transformer的表征能力、RNN的时序建模，在不同场景各司其职。

Tesla Autopilot FSD Chip上，BEV+Transformer推理延迟 <10ms（8相机输入），而量产车型仍广泛保留CNN backbone以平衡功耗与延迟。tesla.com/AI — Tesla AI Day

发现三：团队组织正在结构性重构——Forward Deployed Engineer成为AI原生公司标配

Scale AI首创的FDE（Forward Deployed Engineer）模式已被OpenAI、Anthropic、Cohere、Mistral等公司复制——工程师嵌入客户现场，从"写代码"转变为"写spec定义问题"。Anthropic以约5000人支撑$30B ARR，CEO仅1个直接下属，System Designer/Implementer 50/50分工是核心组织创新。洞察： intelligence-to-headcount比率已永久性改变，PM需要能交付简单功能。

Scale AI的FDE模式被OpenAI/Anthropic/Cohere/Mistral复制，Forward Deployed Engineer是2026年AI公司增长最快岗位。getperspective.ai — Scale AI Forward Deployed Engineers

发现四：MLOps全生命周期=持续训练+金丝雀+回滚——三环缺一不可

Netflix每晚全量重训练+每5分钟在线更新+金丝雀<30秒回滚，是生产ML系统的最佳实践模板。Meta推荐系统每15-60分钟增量更新Embedding层，每天全量重训练排序模型，增量更新后AUC与全量训练差异<0.5%。核心原则：Champion模型全程热备，回滚是流量配置变更而非重新部署。

Google MLOps三级成熟度模型：Level 0（手动）→ Level 1（持续训练CT）→ Level 2（CI/CD流水线自动化）。cloud.google.com — MLOps Continuous Delivery

发现五：推理架构从"单点优化"进化到"系统级协同"——vLLM/Paged Attention成LLM推理事实标准

TensorRT-LLM、ONNX Runtime、vLLM三分天下推理框架市场。vLLM的Paged Attention通过分页内存管理将LLM推理吞吐提升24倍，已被OpenAI、Anthropic、字节豆包广泛采用。KServe+Istio成为Kubernetes上模型推理的标准组合，支持灰度/A-B/Canary多种发布策略。

NVIDIA TensorRT-LLM相比CPU可提升最高36倍推理性能，支持FP8量化和张量并行。developer.nvidia.com — TensorRT

发现六：国产AI基础设施全栈崛起——昇腾/昆仑/PAI已具备规模生产能力

华为昇腾910B（对标H800）、Atlas 900 SuperPOD、MindSpore自动并行；百度昆仑芯3代（256 TFLOPS）、百舸异构计算平台；阿里PAI-EPL弹性并行；字节10,000+ H800/H100集群、火山引擎ML平台。核心能力：国产集群已从"可用"进入"好用"阶段，故障自动恢复、30天不中断训练已成国内厂商标配。

2.1.4 Tesla Dojo — 晶圆级定制路线

Tesla Dojo采用与NVIDIA/Google完全不同的领域定制路线：

  • D1 Chip：362 TFLOPS（FP16），354核定制CPU，片内带宽10TB/s，功耗约500W TDP
  • 晶圆级封装：每个Dojo Cabinet含12个节点（12×D1），算力4,344 TFLOPS（FP16），通过2D Mesh Torus网络互联，延迟<1微秒
  • 无DRAM设计：利用大规模SRAM片上存储（约440GB SRAM per WSE），消除外部显存带宽瓶颈
  • 训练收益：Tesla AI负责人Andrej Karpathy披露，Dojo单节点训练效率约为A100的4倍（特定workload）
  • 应用：训练FSD感知模型，处理每秒数百万帧视频数据；Dojo超算提供1.8 EFLOPS算力
Tesla Dojo D1芯片采用晶圆级封装，片内带宽10TB/s，是自动驾驶AI训练的专用超算。tesla.com/AI — Tesla AI Day

四、经典模型生产运维（CNN/RNN/LSTM）

4.1 Tesla Autopilot — CNN架构三次大迭代

第一代 HydraNet（2019）：多任务CNN

  • RegNetY-64GF backbone + 8个独立heads（目标检测/车道线/可行驶区域等）
  • 特征缓存机制：每帧特征提取结果复用，降低重复计算开销
  • 多任务联合训练，避免孤岛模型问题

第二代 BEV+Transformer（2021-2022）

  • 8相机构成360°视野，通过Transformer融合到统一3D BEV空间
  • ViT替代RegNet作为backbone；Occupancy Network替代2D检测头
  • FSD Chip推理：BEV+Transformer延迟 <10ms（8相机输入）
  • mAP相比2D检测提升约12%

第三代 端到端大模型（2023-2024）

  • 取消BEV直接输出规划，>1亿参数（vs HydraNet约5000万参数）
  • Dojo超算1.8 EFLOPS算力训练
  • 量产车型CNN backbone仍广泛存在——功耗/延迟/稳定性的最优解
Tesla Autopilot从HydraNet演进到BEV+Transformer，量产车型仍保留CNN backbone，混合架构是生产现实。arXiv:2003.10755 — HydraNet

4.2 Google Translate — LSTM→Transformer三年迁移

Google Translate从2016年LSTM迁移到Transformer，采用了三年渐进式混合架构策略：

  • Phase 1（2017-2018）：Transformer Encoder + LSTM Decoder混合架构
  • Phase 2（2018-2019）：全Transformer架构
  • 最终收益：BLEU 24.3→26.5（+9%），延迟降低47%
Google LSTM到Transformer渐进迁移，BLEU+9%，延迟-47%；保留了LSTM Decoder作为小语种翻译快速回滚方案。ai.googleblog.com — Transformer Translation

4.3 Meta推荐系统 — 500亿次/天DNN+Embedding

  • 日均请求：500亿次推荐

  • 在线模型：1,000+个推荐模型，每个有独立SLO
  • 两阶段架构：Candidate Generation（召回数千候选）+ Ranking（DNN+Embedding精排）

  • 金丝雀标准流程：Shadow Mode(24h)→0.1%→1%→5%→25%→50%→100%，每阶段有强制观察期
  • Guardrail红线：p99延迟<2x Champion；Error Rate<0.5%
  • 回滚机制：Champion全程热备，回滚=流量配置变更，<30秒完成
  • A/B测试：回答"更好吗"（Superiority），需p-value<0.05（95%置信度）；Netflix在5%流量下需7-14天达到显著
Kayenta（Netflix开源）支持自动化金丝雀分析，集成Prometheus/Datadog指标，支持自动回滚。github.com/Netflix/kayenta

5.1 OpenAI GPT系列训练方法论

GPT-3（2020）：纯预训练

  • 规模：175B参数，3000亿tokens训练数据
  • 架构：96层Transformer，8K上下文
  • 训练成本：约3.6M A100 GPU小时，估算约$460万

InstructGPT（2022）：RLHF三阶段

  • SFT：人工标注demonstration数据微调预训练模型
  • RM：训练奖励模型（pairwise comparison数据）
  • PPO：强化学习对齐，RLHF让模型更符合人类意图
InstructGPT提出SFT+RM+PPO三阶段RLHF范式，模型规模越大RLHF收益越高。arXiv:2207.09271 — InstructGPT

GPT-4（2023）：技术细节未公开

  • OpenAI几乎未披露技术细节，仅确认使用RLHF和"对抗测试"
  • 据估算训练成本超过$1亿美元
  • 采用"预测缩放定律"（Prediction-Based Scaling Laws）指导训练

5.2 Google PaLM训练

  • 规模：540B参数，6144块TPU v4，7800亿tokens
  • Pathways架构：跨4个TPU Pod（每Pod 4096片）统一调度
  • 训练稳定性：bfloat16混合精度 + Gradient Checkpointing + RoPE位置编码
  • 训练成本：估算>$10M
PaLM 540B使用6144 TPU v4训练，7800亿tokens，Pathways跨Pod统一调度。arXiv:2204.02311 — PaLM

5.6 训练稳定性关键技术

  • Gradient Checkpointing：重计算激活值换取显存节省，可将显存需求降低50%+
  • ZeRO-3：参数分片，线性降低显存占用
  • FSDP（FullyShardedDataParallel）：PyTorch原生，与ZeRO-3等价
  • bfloat16混合精度：比FP32省50%显存，稳定性优于FP16

6.2 模型全生命周期关键组件

  • Orchestrator（编排器）：流水线调度（Kubeflow Pipeline/MLflow/Lightwood）
  • Data Validator：数据质量门禁（TFDV/Great Expectations）
  • Feature Store：特征存储与复用（Feast/Hopsworks/Tecton）
  • Model Registry：模型版本元数据管理（MLflow Model Registry/SageMaker Registry）
  • Experiment Tracker：实验记录（MLflow/W&B/Neptune）

6.3 持续训练（Continuous Training）触发机制

  • 数据漂移触发：PSI>0.2，自动触发Model Retraining Pipeline
  • 定时触发：每日/每周全量重训练（如Netflix每晚）
  • 性能下跌触发：AUC下跌>1%触发告警，人工确认后重训练
  • 新数据可用触发：数据仓库新分区写入触发ETL→训练→部署流水线

6.4 模型版本管理与回滚

  • MLflow Model Registry：四阶段流转（Staging→Production→Archived）+ API驱动
  • Champion热备原则：始终保留最近3-5个可回滚版本，不删除旧版本
  • 回滚=<30秒：Istio/Envoy流量网关调整权重，无需重新部署
  • 版本标签化：所有模型版本打入日期-commit hash标签，存入Model Registry
