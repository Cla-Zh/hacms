模块一：产业界全景图谱（巨头与买家）

AI大模型训练与推理对存储系统提出了前所未有的挑战：万亿参数模型的Checkpoint写入需要TB/s级带宽、推理KV Cache需要GPU显存以外的容量扩展、多模态训练需要小文件高并发吞吐。以下从三个维度盘点全球最核心的20+巨头。

1.1 算力与芯片提供商

🟢 NVIDIA — 全栈AI算力帝国的存储战略

AI存储技术路线：

  • GPUDirect Storage (GDS)：允许NVMe存储设备绕过CPU，通过RDMA直传数据至GPU显存，消除CPU拷贝瓶颈。当前GDS 2.x版本支持CuFile API，在DGX H100上可实现28GB/s单GPU读取带宽。NVIDIA GPUDirect Storage Design Guide, 2026
  • Vera Rubin R100 + HBM4：336亿晶体管，288GB HBM4，单GPU内存带宽22TB/s（相比HBM3e的8TB/s提升2.75倍）。这是解决"内存墙"的硬件级方案。NVIDIA Investor Relations, CES 2026
  • CPO共封装光学：将光收发器封装在GPU封装内，NVLink 6带宽达1.8TB/s双向，机柜级全光互联。Xueqiu, 英伟达Rubin平台CPO技术标准, 2026.5
  • Groq 3 LPU集成：$200亿收购Groq后，将LPU推理架构集成至Vera Rubin平台，实现训练+推理异构协同。CNBC, Dec 2025

痛点：GPU算力增速（2x/代）>HBM带宽增速（1.5x/代），"内存墙"持续恶化；GDS对应用透明度不足，需改造数据加载Pipeline。

🟢 AMD — UALink挑战NVLink的开放生态

  • MI350X/MI400系列：CDNA 4架构，MI455X搭载HBM4，UALink over Ethernet（UALoE72）互联72颗GPU，打破NVIDIA NVLink封闭生态。AMD CDNA 4 Architecture White Paper, March 2026
  • Helios AI Rack：72颗MI455X + Zen 6 EPYC Venice，31TB共享显存，CES 2026公布。TechTimes, CES 2026 AMD Helios
  • Infinity Fabric：节点内高速互联，但跨节点仍依赖以太网/RoCE，与NVIDIA NVLink over Fabric有差距。

痛点：ROCm生态仍远落后于CUDA；MI400量产时间（2026下半年）晚于Vera Rubin。

🟢 Intel — Habana Gaudi + CXL推手的双重角色

  • Habana Gaudi3：AI训练加速器，但市场接受度有限，Intel已战略收缩。
  • CXL核心推动者：Intel是CXL规范的发起者，Sapphire Rapids/Granite Rapids原生支持CXL 2.0 Type3内存扩展。但SemiAnalysis 2024年著名文章" CXL Is Dead in the AI Era "指出：AI训练中GPU-Direct路径更高效，CXL在AI训练场景的价值有限，主要适用于推理KV Cache扩展。SemiAnalysis, "CXL Is Dead in the AI Era", March 2024
  • Intel GNR + CXL 3.0：Granite Rapids-AP支持CXL 3.0，可做CXL Switch实现多节点内存池化。CSDN, "CXL内存池化服务器架构革命", 2026

3.3 重点深度拆解：10篇最具颠覆性论文

🏆 #1 Mooncake (FAST'24 最佳论文)

全名：Mooncake: A KVCache-centric Disaggregated Architecture for LLM Serving

机构：清华大学（章明星、武永卫、郑纬民团队）+ 月之暗面(Moonshot AI)

底层逻辑：传统LLM推理中，Prefill（长上下文编码）和Decode（逐Token生成）对GPU资源的需求完全不同。Mooncake的核心洞察是——KV Cache是推理架构的"一等公民"，而非GPU显存的附属品。

  • 架构拆解：Prefill集群（高算力GPU）与Decode集群（低算力GPU）物理分离，中间通过"Volatile Buffer"传输KV Cache
  • 存储创新：KV Cache从GPU显存卸载至主机DRAM+SSD，实现跨请求复用。未被使用的CPU/SSD资源成为"免费"的KV Cache存储
  • 核心效果：在Kimi生产环境中，支持4x并发用户，首Token延迟降低50%+

FAST'24 Erik Riedel Best Paper Award; Sina News; alphaxiv.org/overview/2407.00079v4

🏆 #2 GCR (FAST'26 杰出技术贡献奖)

全名：GPU Checkpoint/Restore Made Fast and Lightweight

机构：清华大学存储实验室

底层逻辑：AI训练中Checkpoint写入是最大I/O瓶颈——万亿参数模型一次Checkpoint可达数十TB。传统方案将GPU显存→CPU内存→存储的路径视为黑盒，GCR的关键洞察是分离"数据路径"和"控制路径"。

  • 数据路径：CUDA直接DMA传输GPU显存数据至RDMA目标，绕过CPU拷贝
  • 控制路径：轻量级元数据管理，仅记录数据位置映射
  • 核心效果：Checkpoint速度提升数倍，Restore从分钟级降至秒级

FAST'26 Distinguished Artifact Award; Tsinghua CS News March 2026; usenix.org/system/files/fast26-zeng.pdf

🔬 #3 InstInfer (arXiv 2024)

全名：InstInfer: In-Storage Attention Offloading for Cost-Effective LLM Serving

底层逻辑：将注意力计算（Decode阶段的性能瓶颈）直接卸载至SSD内部执行。SSD控制器内置计算单元处理KV Cache的注意力运算，消除GPU↔SSD之间海量数据搬运。这意味着SSD不再是"哑存储"，而是"会算的存储"。

arXiv:2409.04992

🔬 #4 Beluga (arXiv 2025)

全名：Beluga: A CXL-Based Memory Architecture for Scalable and Efficient LLM Inference

底层逻辑：系统性刻画CXL内存池的性能特征（带宽/延迟/拓扑），提出Beluga-KVCache系统设计。核心发现：CXL内存池的延迟(~150-300ns)远低于RDMA+SSD(~10μs)，但高于本地HBM(~50ns)，因此KV Cache需要三级分层：HBM→CXL→SSD。

arXiv:2511.20172

🔬 #5 DataStates-LLM (ISC'24)

全名：DataStates-LLM: Lazy Asynchronous Checkpointing for Large Language Model Training

底层逻辑：利用CUDA异步操作+Linux io_uring实现Checkpoint I/O与训练Forward/Backward的重叠。关键创新是"懒惰"策略——不立即写Checkpoint，而是在GPU计算空闲期异步刷出。实现高频Checkpoint（每100步而非每1000步）而几乎不影响MFU。

ACM DL 10.1145/3625549.3658685; github.com/DataStates/datastates-llm

🔬 #6 PolarCXLMem (SIGMOD'25)

全名：Unlocking the Potential of CXL for Disaggregated Memory in Cloud-Native Databases

底层逻辑：阿里云+Intel合作，在PolarDB上验证CXL分离内存。核心发现：CXL内存作为远端内存池可扩展至数TB，延迟在可接受范围，但需要数据库引擎感知CXL的NUMA拓扑特性。这是CXL在云数据库生产环境的首次系统性评估。

SIGMOD 2025; modb.pro/doc/145811

🔬 #7 CXLAimPod (arXiv 2025)

全名：CXLAimPod: CXL Memory is All You Need in AI Era

底层逻辑：提出自适应调度框架，通过cgroup hint告知操作系统哪些内存区域适合放置在CXL远端内存上。弥合CXL硬件能力与软件栈之间的鸿沟——当前OS/应用对CXL内存的感知几乎是零。

arXiv:2508.15980

🔬 #8 PagedAttention (SOSP'23)

全名：Efficient Memory Management for Large Language Model Serving with PagedAttention

底层逻辑：将操作系统虚拟内存管理思想应用于KV Cache——KV Cache不需要连续显存，按"页"管理，按需分配/释放。这是vLLM的基础，解决了KV Cache碎片化导致的显存浪费问题，使推理吞吐量提升2-4x。

SOSP 2023; vLLM project

🔬 #9 AiF (ISCA'25)

全名：AiF: In-Flash Attention for LLM Inference

底层逻辑：在闪存芯片内部执行GEMV（通用矩阵-向量乘法）运算。不再将KV Cache数据从SSD搬至GPU，而是在SSD端完成注意力计算，仅将结果传回GPU。这是"存算一体"在AI推理场景的最前沿探索。

ISCA 2025

🔬 #10 Splitwise (ISCA'24)

全名：Splitwise: Efficient Generative LLM Inference Using Phase Splitting

底层逻辑：与Mooncake独立提出的类似洞察——Prefill和Decode应使用不同的GPU配置。但Splitwise从芯片架构层面分析：Prefill需要高算力+低带宽GPU，Decode需要低算力+高带宽GPU。提出了混合GPU集群的设计方法论。

Microsoft Research, ISCA 2024

模块四：大胆的未来预测与架构演进（3-5年）

🔮 预测1：AI集群将走向"存算分离2.0"——但不是传统意义上的分离（2027-2028）

当前的"存算分离"争论陷入二元对立：要么一切在GPU内（NVIDIA路线），要么CXL池化一切（Intel路线）。未来3年的赢家将是"语义感知的分层存算分离"：

  • 热数据层：HBM4/4E（~50ns延迟）——活跃KV Cache + 模型权重
  • 温数据层：CXL内存池（~200ns延迟）——长上下文KV Cache + Checkpoint缓冲
  • 冷数据层：NVMe SSD + 计算存储（~10μs延迟）——训练数据 + 归档Checkpoint

这不是简单的"存储在远处"，而是每一层都具备计算能力：HBM做Decode，CXL做KV Cache管理和Prefill部分计算，SSD做注意力卸载(AiF/InstInfer)。这将使单GPU有效推理容量扩展1000x。

置信度：75% | 依据：Mooncake生产验证、Beluga系统设计、XConn+MemVerge OCP演示

🔮 预测2：硅光互连将在2028-2029年全面替代GPU集群铜缆InfiniBand

当前AI集群的互联瓶颈已从"单节点内"转向"跨节点"：NVLink 6可达1.8TB/s，但节点间仍依赖InfiniBand/Ethernet（~400-800Gb/s）。硅光互连的革命性在于：

  • Celestial AI Photonic Fabric：10x带宽+10x延迟降低+4-8x功耗改善
  • Ayar Labs UCIe光Chiplet：封装内光学I/O
  • NVIDIA CPO（Vera Rubin）：机柜级光学互联的首次大规模部署

但硅光全面替代铜缆的转折点不在技术，而在成本曲线。预计2028年硅光模块成本降至$1/Gb以下，届时InfiniBand的$/Gb成本优势将被翻转。

置信度：60% | 依据：Celestial AI $515M融资、NVIDIA CPO已在Vera Rubin、Ayar Labs UCIe标准

🔮 预测3：HBM将在2029年前后触及物理极限，3D SRAM/PIM将接棒

HBM的堆叠层数从8-high→12-high→16-high，但每层厚度减薄至~30μm后面临热管理极限。SK Hynix的HBM4E（12-high 48GB）可能是传统TSV堆叠的巅峰。下一代方案：

  • 3D SRAM + 逻辑Die混合堆叠：SRAM延迟~0.5ns vs DRAM ~50ns，但面积效率低
  • d-Matrix式DIMC：SRAM存内计算消除数据搬运，150TB/s带宽
  • Hybrid Bonding：铜-铜混合键合取代微凸点，互连密度提升10x

这意味着2029年前后的AI芯片将不再是"GPU+HBM"的模式，而是"逻辑Die+3D SRAM+3D DRAM"的混合堆叠。

置信度：55% | 依据：HBM4E路线图、d-Matrix Hot Chips 2025、KAIST HBM报告371页

🔮 预测4：LLM推理架构将根本性重构——KV Cache将成为独立的基础设施服务（2027-2028）

当前LLM推理的KV Cache管理是"嵌入在推理引擎中的辅助功能"。未来它将演变为独立的基础设施服务：

  • KV Cache Store：类似Redis的独立KV Cache服务，跨推理实例共享
  • KV Cache CDN：多地域部署KV Cache，用户请求路由至最近节点
  • KV Cache Tiering Engine：自动决定哪些KV Cache在HBM/CXL/SSD

Mooncake已证明KV Cache可独立于推理引擎存在，LMCache和SGLang的Hierarchical KV Cache正在标准化这一接口。2028年，"KV Cache即服务"将成为云厂商的标准产品。

置信度：70% | 依据：Mooncake生产部署、LMCache开源、WEKA NeuralMesh Token Warehouse

🔮 预测5：AI训练的Checkpoint将从"存储问题"变为"网络问题"（2027-2028）

当前Checkpoint优化的核心是"如何快速写入存储"（GCR、DataStates-LLM）。但随着模型规模从万亿→十万亿参数，单次Checkpoint数据量从数十TB→数百TB，瓶颈将从存储带宽转移到网络带宽：

  • 20万GPU集群同时Checkpoint：瞬间需要数百TB/s的聚合网络带宽
  • 当前InfiniBand NDR 400Gb/s/port远不够，需要1.6Tb/s/port+
  • 解决方案：分布式异步Checkpoint（不同GPU组错峰写入）+ 网络拓扑感知调度

Meta的SC'25论文已开始探索拓扑感知Checkpoint，这是从"存储优化"转向"网络+存储联合优化"的信号。

置信度：65% | 依据：Meta SC'25、xAI Colossus 20万GPU规模、NVIDIA CPO网络带宽规划
