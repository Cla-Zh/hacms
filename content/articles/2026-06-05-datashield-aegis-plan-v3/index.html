ch3 Aegis Agent

3.1 命名与定位

Aegis（宙斯盾）— 主动防御·不可穿透·预判守护

不叫"数据安全Agent"——Aegis含义远超"安全"：

  • 预判(Oracle)：不等攻击发生，而是预判攻击将至
  • 守护(Aegis)：不是检测告警，而是主动打快照、隔离、恢复
  • 审判(Arbiter)：不是简单阈值，而是LLM驱动智能决策
  • 浴火(Phoenix)：不是简单还原，而是确保数据干净的恢复
  • 铭史(Chronicler)：不是简单日志，而是因果链取证审计

3.2 七类Agent

Sentinel 哨兵

4路数据采集(100ms,5s聚合)，纯特征工程，不依赖LLM

Oracle 预言者

调用DataShield LLM预判，输出风险评分0-100，核心差异化：攻击前30s+预判

Arbiter 仲裁者

综合预判+业务+策略，决定L0-L4防护级别

Aegis 守护者

执行防护(快照/只读/隔离/Air Gap)，L3时30s超时自执行

Phoenix 浴火者

验证快照/备份干净性，LLM二次验证恢复数据

Chronicler 铭史者

因果链取证+ATT&CK映射+时间线重建

Scholar 学者

从每次事件中学习，更新Oracle模型，数据飞轮闭环

3.3 核心创新：攻击前预判快照

★ RPO≈0的秘密

  1. Oracle在IO模式/熵值趋势出现攻击前兆时，提前30s+发出预判
  2. Aegis在攻击确认前触发快照(数据100%干净)
  3. 渐进防护：L1(50-70)只打快照→L2(70-85)只读→L3(85-95)隔离→L4(95-100)Air Gap
  4. 恢复时使用"攻击前预判快照"，数据零损失
