1概述 (Overview)

D-Series AI Card 是面向 AI 训练/推理场景的智能 NPU 资源池化系统, 核心解决 2 个问题:

  1. 多卡池化 (Multi-card Pool) — 1 个逻辑 D-Card = N 张物理 NPU, 异构卡混部
  2. 容器化 (Container Integration) — 跟 K8s 无缝集成, 容器秒级看到池化后的资源

D1 v3.0 新能力

v3.0 新增 3 大服务: AI 推理服务 (InferenceService) 暴露 5 endpoint 包含 Init 自动加载模型 + 推理 + 资源管理; 健康管理服务 (HealthService) 4 endpoint 含实时流式监控 + Prometheus 指标; CANN 自初始化 (cann-init.sh) 容器启动 hook 自动初始化 CANN 8.0.RC2 框架 + 扫描 /opt/dcard/models/ 加载 .om 模型. 默认 vendor 从 NVIDIA 切换到华为 Ascend 910B/310P + CANN 8.0.RC2 + HCCS 集合通信.

1.1 D1 v3.0 范围 (Q3 2026)

  • 8 卡 Ascend 910B 池 (1 逻辑 = 8 物理, HCCS 集合通信)
  • vendor 字段在 K8s 可见 (vendor=ascend)
  • 容器化训练 + 推理任务秒级启动
  • 推理服务自动初始化 CANN + 加载 /opt/dcard/models/*.om ✅
  • 健康检查 (Check / Watch 流式 / IngestMetrics / GetCardHealth) ✅
  • 故障 1s 检测 + 5s 隔离 (基础, 不做自动迁移)

1.2 D1 v3.0 不做

  • ❌ 动态扩缩 (D2 Pool Manager)
  • ❌ 故障自动迁移 (D3 Hot-Swap)
  • ❌ 跨集群 (D4 Federation)
  • ❌ 边缘 + 混合云 (D5)
  • ❌ 推理调度 (QPS 路由 / 模型版本 / A/B Test)

3架构总览 (Architecture)

D1 系统由 5 大子系统 组成, 自底向上分 3 层.

3.1 3 层职责

  • Layer 1 硬件抽象: D-Card Mgr + PCIe HAL, 处理物理 NPU 资源, 通过 HCCS 互联
  • Layer 2 池化抽象 (D1 简化): dctl CLI 运维接口 (D1 不做编排)
  • Layer 3 K8s 集成: Device Plugin (基于 ascend-device-plugin) + Runtime Shim (基于 ascend-docker-runtime)

v3.0 新增层

Layer 1 与 Layer 3 之间新增 InferenceService + HealthService 两个 gRPC 服务, 暴露在每个 K8s 节点的 /var/run/dcard.sock Unix Domain Socket, 供业务容器 (推理容器 / 监控容器) 调用. CANN 自初始化 (cann-init.sh) 在容器启动时通过 Runtime Shim 触发.

4池化模式 (Pooling Modes)

D1 v3.0 支持 3 种池化模式, 适用不同场景:

模式适用卡型原理1 物理 = N 逻辑性能损失
硬切片 NPU SliceAscend 910B / 910CNPU 内置硬切, 7 个独立实例1 物理 = 7 NPU Slice0% (硬件隔离)
软切片 vNPUAscend 310PCANN 软切分 (类似 vGPU Manager)1 物理 = 2-8 vNPU5-15% (驱动开销)
HCCS 互联Ascend 910B (主推)Huawei Cache Coherent Interconnect 多卡拓扑1 物理 = 2-56 HCCS 切片<5% (硬件辅助)

D1 v3.0 默认模式: HCCS 互联 (Ascend 910B + CANN 8.0.RC2). D1 不做动态切换, 需重启 Pool.

10部署架构 (Deployment Architecture)

D1 v3.0 部署在 K8s 集群每个节点, 涉及 3 个核心组件 + 新增 2 个 gRPC 服务 + 1 个启动 hook.

10.1 部署要求

  • 每个 K8s 节点装 D-Card Mgr + PCIe HAL DaemonSet
  • 8 卡 Ascend 910B 节点 = 1 节点 1 Pool (8 物理 = 1 逻辑 = 8 NPU Slice, 或 56 NPU 切片细粒度)
  • 容器通过 /dev/davinci* 设备访问 (Ascend 910 标准设备名)
  • v3.0 新增: 节点暴露 InferenceService (端口 50051) + HealthService (端口 50052) gRPC Unix Domain Socket

11风险与缓解 (Risks & Mitigations)

#风险影响缓解
1NPU 切片性能差异同卡不同 NPU Slice 实例性能不一致D1 限制: 1 物理卡只暴露 1 个 NPU Slice (性能优先)
2PCIe 带宽争用多容器共享 PCIe 带宽D1 限制: 1 Pool 内 1 容器独占 PCIe (D2 调度优化)

4HCCS 拓扑压测多卡 HCCS 高速互联需充分压测 (国产卡主推的代价)D1 8 卡 ring + mesh 双拓扑, D2 升级到 full-mesh
5K8s Device Plugin 重启期间 Pod 调度失败D1 容忍 30s 重启窗口 (kubelet 重试)
6故障检测误报HCCS 心跳偶发超时误判 FAILEDD1 用 3 次连续失败才标记 FAILED (防抖)

15AI 推理服务 (InferenceService) v3.0 新增

AI 推理服务暴露 5 个 gRPC endpoint, 监听在 0.0.0.0:50051 (容器内) 或 unix:///var/run/dcard-inference.sock (节点 DaemonSet), 负责推理全生命周期管理.

15.1 完整 Protobuf 定义

// dcard/inference/v1/inference.proto
syntax = "proto3";
package dcard.inference.v1;
import "google/protobuf/timestamp.proto";

service InferenceService {
  // Init: 启动时自动调用, 初始化 CANN + 加载 /opt/dcard/models/
  rpc Init(InitRequest) returns (InitResponse);
  // LoadModel: 运行时热加载模型
  rpc LoadModel(LoadModelRequest) returns (LoadModelResponse);
  // Infer: 单次推理
  rpc Infer(InferRequest) returns (InferResponse);
  // Deinit: 清理 + 释放 NPU
  rpc Deinit(DeinitRequest) returns (DeinitResponse);
  // ListModels: 列出已加载模型
  rpc ListModels(ListModelsRequest) returns (ListModelsResponse);
}

message InitRequest {
  string model_dir = 1;         // 默认 /opt/dcard/models/
  bool auto_load_all = 2;       // true: 扫描目录加载所有 .om
  map<string, string> cann_env = 3;  // CANN 环境变量
  HccsConfig hccs = 4;          // HCCS 集合通信配置
}

message InitResponse {
  int32 loaded_count = 1;       // 加载成功的模型数
  repeated ModelInfo models = 2;
  CannHealth cann_health = 3;   // CANN 进程健康状态
}

message LoadModelRequest {
  string model_path = 1;        // /opt/dcard/models/xxx.om
  string model_name = 2;        // 业务侧命名
}

message LoadModelResponse {
  ModelInfo model = 1;
}

message InferRequest {
  string model_name = 1;
  bytes input_data = 2;         // 序列化输入 (TensorProto / numpy)
  map<string, string> params = 3;
}

message InferResponse {
  bytes output_data = 1;
  int64 latency_us = 2;         // 推理耗时 (微秒)
  InferState state = 3;
}

message DeinitRequest {
  bool release_npu = 1;         // true: 释放 NPU 资源
}

message DeinitResponse {
  bool released = 1;
}

message ListModelsRequest {}
message ListModelsResponse {
  repeated ModelInfo models = 1;
}

message ModelInfo {
  string name = 1;
  string path = 2;
  int64 size_bytes = 3;

15.2 Init 启动流程详解

Init 自动加载

容器启动时, Runtime Shim 自动调用 InferenceService.Init (而非用户手动调用). Init 内部执行 4 阶段:

  1. (a) 加载 Ascend 驱动 — 通过 modprobe + npu-smi 检查 8 卡
  2. (b) 设置 HCCS 集合通信 — 初始化 hccl 进程, 配置 rank_id / world_size
  3. (c) 扫描 /opt/dcard/models/ — 加载所有 .om 模型, 调用 aclmdlLoadFromFile
  4. (d) 注册到 device-plugin — 暴露 huawei.com/ascend-pool 资源

17CANN 自初始化 (cann-init.sh) v3.0 新增

CANN 自初始化是容器启动时的 hook 脚本, 类比 NVIDIA 的 nvidia-container-runtime. 由 Container Runtime Shim 在 runc create 容器后立即调用.

17.1 cann-init.sh 完整脚本

#!/bin/bash
# cann-init.sh - D1 v3.0 CANN 自初始化 (类似 nvidia-container-runtime)
# 调用时机: Runtime Shim 拦截 runc create 后, 容器进程启动前
# 退出码: 0=成功, 1=驱动加载失败, 2=HCCS 配置失败, 3=模型加载失败

set -e

MODEL_DIR="${DCARD_MODEL_DIR:-/opt/dcard/models}"
AUTO_LOAD="${DCARD_AUTO_LOAD:-true}"
LOG_FILE="${DCARD_INIT_LOG:-/var/log/dcard/cann-init.log}"

mkdir -p "$(dirname "$LOG_FILE")"
exec > >(tee -a "$LOG_FILE") 2>&1

echo "===================================================="
echo "  cann-init.sh - D1 v3.0 CANN 自初始化"
echo "  启动时间: $(date -Iseconds)"
echo "  模型目录: $MODEL_DIR"
echo "===================================================="

# (a) 加载 Ascend 驱动 + 检查 8 卡
echo "[1/4] 加载 Ascend 驱动 + 检查 NPU 设备..."
modprobe drv_pcie_ascend 2>/dev/null || true
modprobe drv_ascend_hal 2>/dev/null || true
DEVICE_COUNT=$(npu-smi info -l | grep -c "Ascend 910B" || true)
if [ "$DEVICE_COUNT" -lt 8 ]; then
    echo "  ❌ 只检测到 $DEVICE_COUNT 张 NPU, 期望 8"
    exit 1
fi
echo "  ✅ 检测到 $DEVICE_COUNT 张 Ascend 910B NPU"

# (b) HCCS 集合通信配置
echo "[2/4] 配置 HCCS 集合通信..."
export ASCEND_HOME=/usr/local/Ascend
export HCCL_CONNECT_TIMEOUT=300
export HCCL_RANK_ID=${HCCL_RANK_ID:-0}
export HCCL_WORLD_SIZE=${HCCL_WORLD_SIZE:-8}
export HCCL_MASTER_ADDR=${HCCL_MASTER_ADDR:-"127.0.0.1:50000"}
echo "  HCCL_RANK_ID=$HCCL_RANK_ID, WORLD_SIZE=$HCCL_WORLD_SIZE"

# (c) 扫描 /opt/dcard/models/ 加载 .om 模型
echo "[3/4] 扫描 $MODEL_DIR 加载 .om 模型..."
if [ ! -d "$MODEL_DIR" ]; then
    echo "  ⚠️  模型目录不存在: $MODEL_DIR, 跳过"
elif [ "$AUTO_LOAD" = "true" ]; then
    MODEL_COUNT=0
    for model_file in "$MODEL_DIR"/*.om; do
        if [ -f "$model_file" ]; then
            MODEL_NAME=$(basename "$model_file" .om)
            echo "  加载: $MODEL_NAME ($model_file)"
            if /usr/local/dcard/bin/load_model --path="$model_file" --name="$MODEL_NAME"; then
                MODEL_COUNT=$((MODEL_COUNT+1))
            else
                echo "  ❌ 加载失败: $model_file"
                exit 3
            fi
        fi
    done
    echo "  ✅ 成功加载 $MODEL_COUNT 个模型"
else
    echo "  AUTO_LOAD=false, 跳过自动加载"
fi

# (d) 注册到 ascend-device-plugin
echo "[4/4] 注册 NPU 资源到 K8s Device Plugin..."
if [ -x /usr/local/Ascend/driver/tools/hccn_tool ]; then
    /usr/local/Ascend/driver/tools/hccn_tool -i 0 -ip ${HCCL_MASTER_ADDR%:*} 2>/dev/null || true
fi
echo "  ✅ ascend-device-plugin 资源注册完成 (huawei.com/ascend-pool)"

echo "===================================================="
echo "  cann-init.sh 完成 (exit 0)"
echo "===================================================="
exit 0
