项目概述

AAM-CMS（AI-Agent Managed Static CMS）是一个专为 AI Agent 内容生命周期管理设计的完全去后端（Zero-Backend）静态知识库系统。

它解决了以下核心问题：

  • 多格式内容（HTML/Word/PPT/PDF）分散管理，难以统一检索
  • 传统 CMS 需要数据库和后端服务，增加运维复杂度和攻击面
  • Agent 在执行任务时缺乏统一的资产管理规范，容易产生\"文件碎片化\"问题

核心架构设计

1. 资产聚合模型

相同主题的不同格式文件（index.html、main.docx、slides.pptx）必须聚合在同一个文章目录下，通过 manifest.json 的 attachments 字段统一索引。

2. 双层分类过滤

系统整合了"父级目录（Category）"与"横向标签（Tags）"的双重筛选逻辑，配合前端模糊搜索实现秒级检索。

3. 零后端架构

纯静态部署，无需数据库。可托管于任意静态文件服务器（ Nginx、GitHub Pages、Vercel、Cloudflare Pages）。

4. Agent 行为约束

通过 .harness/harness.json 对 Agent 行为进行规范约束，确保其：

  • 处理文件前必须先读取 harness 规范
  • 禁止在前端核心架构文件（index.html、app.js、style.css）上进行未授权修改
  • 多格式文件必须聚合存放，manifest.json 中不得出现重复条目

前端功能说明

列表视图（List State）：展示所有文章卡片，支持分类筛选、标签筛选和实时模糊搜索。卡片上用小图标标出该文章包含的格式（HTML、WORD、PPT图标）。

文章框架（Reader Frame）：点击卡片后进入全屏 iframe 框架，左上角常驻 [◀ 返回] 按钮，右上角根据 attachments 动态渲染下载按钮（Word版、PPT演示文稿等）。

目录结构

├── .harness/
│   └── harness.json          # 智能体行为宪法
├── content/
│   ├─ index/
│   │   └── manifest.json     # 全局动态索引
│   ├─ articles/
│   │   └─ [topic-slug]/
│   │       ├─ index.html     # 主体报告（iframe 加载）
│   │       ├─ main.docx      # Word 附件
│   │       └─ slides.pptx     # PPT 附件
│   └── assets/
├── index.html                # 前端主外壳
├── app.js                    # SPA 路由 & 搜索逻辑
├── style.css                 # 高质感样式表
└── README.md

快速开始

本地预览

在项目根目录执行：

python3 -m http.server 8080

然后访问 http://localhost:8080 即可预览。

技术规范

  • 前端：原生 JS（无框架依赖），SPA 状态机管理
  • 样式：CSS Variables + Flexbox/Grid，极简主义设计语言
  • 存储：纯 JSON 文件（manifest.json），无需数据库
  • 部署：Nginx / GitHub Pages / Vercel 均可
  • 协议：Apache 2.0
