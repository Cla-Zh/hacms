一、问题澄清：防泄漏 ≠ 防勒索

1.1 两者本质区别

防勒索关注数据可用性（恢复），防泄漏关注数据机密性（外泄）。前者检测加密行为，后者检测数据外传行为。这是完全不同的技术方向。

1.2 前期报告的问题

前期报告中NetApp Cloud Data Sense、Commvault Threat Scan、Dell Cyber Recovery、Rubrik Radar、Cohesity ThreatScan等方案，核心能力集中在存储内恶意软件扫描（防勒索）、不可变备份与Air-Gap（防篡改）、熵值异常检测（防加密）。这些是数据保护能力，而非数据防泄漏能力。真正的防泄漏UEBA方案来自另一类厂商。

二、UEBA防泄漏技术框架

2.1 UEBA定义演进

Gartner已将UEBA从独立市场类别重新归类为Insider Risk Management (IRM)的核心能力。这意味着UEBA不再被视为独立产品，而是内嵌于内部风险管理平台中的关键分析引擎。

三、Exabeam：Smart Timeline + 累积风险评分

3.1 核心算法：累积风险评分

Exabeam的风险评分不是二元判定（正常/异常），而是累积式评分：

1. 基线建立：为每个用户/实体建立"正常行为"基线，包括VPN使用习惯、上班时间、访问系统、邮件附件大小/频率、USB使用等

2. 偏差加分：行为偏离基线时增加风险评分，偏差越大加分越多

3. 累积触发：多个轻微异常累积超过阈值才告警，避免单一误报触发

4. 时间衰减：历史异常影响随时间递减，防止分数只增不减

评分公式（推导）：R(t) = Σ(wi · di · e^(-λ(t-ti)))，其中wi为行为类型权重，di为偏差程度(z-score)，λ为时间衰减系数。

关键特性：单一轻微异常不会触发告警；必须出现多个异常迹象才会告警；上下文感知：海外员工凌晨3点登录不加分（在其基线内）。

【Exabeam, "UEBA Complete 2025 Guide", www.exabeam.com, 2025】

四、Varonis：数据中心型UEBA

4.1 核心技术：权限-行为差异检测

Varonis的核心理念：权限 ≠ 行为。用户拥有访问权限不代表会实际访问。

1. 权限映射：构建"谁可以访问什么"的完整权限图谱

2. 行为追踪：记录"谁实际访问了什么"的行为日志

3. 差异计算：对比权限与行为的差异，识别过度权限

4. 风险评分：过度权限 + 敏感数据访问 = 高风险

五、Securonix：SIEM原生高级UEBA

5.1 同侪分组分析（核心差异化）

同侪分组分析是Securonix的核心差异化算法，也是其AI/ML评分4.0/5的关键支撑。

1. 特征提取：从日志中提取用户/实体行为特征向量（访问系统集合、活动时间分布、数据访问类型分布、网络行为特征）

2. 聚类分组：DBSCAN/K-Means/层次聚类，在多维行为特征空间中自动分群

3. 基线建立：对每个同侪组计算集体行为基线（活动频率分布、访问系统范围、时间模式）

4. 异常判定：用户行为与同侪组基线对比，偏差超过阈值(z-score>2)标记为异常

5.2 多维风险评分

总体风险评分：R = Σ(wi · si · e^(-λΔt))，其中si为各维度归一化子分数(0-1)，λ为时间衰减系数。

【Securonix, "Peer Group Analytics Technical Guide", 2025; "Risk Scoring Algorithm Whitepaper", 2025】

6.2 行为DLP

传统DLP检测"数据是什么"（内容特征）→ 误报率高；行为DLP检测"行为是否有风险"（行为模式）→ 误报率低。DTEX行为DLP自动为文件建立画像，基于创建者、受众、时间投入、谱系(lineage)等因素，无需预定义内容规则即可识别文件是否为敏感资产。

元数据最小化原则：仅收集元数据（非原始内容），隐私合规，高保真精度减少误报。

9.2 Deep Autoencoder + Doc2Vec框架

Fuentes, Ortega-Fernandez, Villanueva, Sestelo, "Cybersecurity threat detection based on a UEBA framework using Deep Autoencoders", AIMS Mathematics, 2025. DOI: 10.3934/math.20251043

架构：原始行为日志 → Doc2Vec嵌入 → Deep Autoencoder → 重建误差 → 异常分数 → SHAP+LIME+Counterfactual三重可解释性

理论保证：基于万能逼近定理（Universal Approximation Theorem），深度自编码器可逼近任意连续函数。三重可解释性：SHAP（全局+局部特征重要性）、LIME（局部线性近似）、Counterfactual（反事实解释"需要改变什么才不是异常"）。

9.3 LSTM Autoencoder

Malhotra et al., ACM ICMLA 2020, DOI: 10.1145/3406601.3406610。架构：行为序列→LSTM Encoder→瓶颈向量z→LSTM Decoder→重建序列→重建误差=异常分数。LSTM天然适合处理时间序列行为数据，能捕获长期依赖关系。

十三、核心发现与建议

1. 防泄漏 ≠ 防勒索：前者关注数据机密性（UEBA），后者关注数据可用性（备份恢复），检测方法完全不同

2. 存储厂商UEBA能力为零：NetApp/Commvault/Dell/Rubrik/Cohesity的DLP能力集中在内容检测和防勒索，UEBA行为分析需依赖第三方

3. Exabeam和Securonix UEBA能力最强：前者以Smart Timeline和累积风险评分著称，后者以同侪分组分析见长

4. DTEX意图三分类是范式创新：将异常从二元（正常/异常）扩展为三元（恶意/疏忽/受损），差异化响应策略

5. Deep Autoencoder+Doc2Vec是学术前沿：2025年AIMS Mathematics论文提出的可解释UEBA框架，结合重建误差检测和SHAP/LIME/Counterfactual三重可解释性
